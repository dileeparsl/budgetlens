# RSL Mini-Hack '26 — Official use case

**Source**: `Hackathon Use Case - RSL Mini-Hack '26.pdf` (March 25, 2026, 90 minutes)

## Problem statement

Build a **simple personal finance tracking** web application that helps users
understand spending and make better money decisions.

## Functional requirements

1. **Transactions** — Users manually enter **income** and **expenses** through
   clear forms/UI.
2. **Categories** — Users create categories (e.g. food, transport, bills,
   entertainment) and assign expenses (and income if you choose) to them.
3. **Summary** — Show **total income**, **total expenses**, and **remaining
   balance**. Support defining **monthly income**, **allocated budget**, and
   **savings targets** (exact UX is up to you).

## Technical requirements

- **Identity** — Simple user identification (login, account, or PIN).
- **Responsive** — Works well on mobile browsers.
- **Persistence** — Store financial records in a **database**.
- **CRUD** — Create, read, update, and delete **transactions** (and categories
  as needed).
- **Landing page** — Useful snapshot: e.g. **current month expenses**,
  **percent of monthly budget used**, **breakdown by category**.
- **Dashboard** — **Charts or graphs** (e.g. monthly spending, category mix,
  historical trends).

## Optional extensions (nice-to-have)

- Insights like “spending more than usual on dining this month”.
- Savings ideas from patterns.
- Flag unusual or unnecessary expenses.
- Basic savings or budget recommendations.
- Natural-language questions (e.g. “Where did I spend the most last month?”).

## Deliverables (reminder)

1. Working app on **free hosting** with **working data storage** + URL (and
   login details if needed).
2. **GitHub** repo link.
3. **AI workflow documentation** (`docs/ai-workflow.md` or shared doc).
